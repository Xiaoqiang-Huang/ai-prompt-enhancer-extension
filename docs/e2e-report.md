# E2E 验证摘要

最近一次本地验证：2026-07-14

## 环境边界

- 使用真实构建产物 `dist` 加载为 unpacked Chrome 扩展。
- 使用本地 fixture 页面，不调用真实 OpenAI、Anthropic、Gemini、DeepSeek 或其他第三方 AI API。
- 使用本地 mock Provider 响应验证增强工作流。
- 不包含 API Key、Cookie、邮箱或用户对话数据。

## 已验证流程

- Popup、Options、Side Panel 可以打开。
- 支持站点 fixture 输入框旁显示绿色小球和次级入口。
- 入口不遮挡普通输入框；宽输入框会自动使用上方/下方布局。
- 增强过程中显示进度，对比视图显示原始提示词和优化结果。
- 直接增强同时返回意图摘要和关键缺失信息；只有检测到关键信息缺失时才显示“进入追问模式”建议。
- 接受前原文不变，拒绝后原文保留，接受后才替换。
- 追问模式会携带用户回答继续增强。
- 继续优化会携带原始提示词和上一轮结果。
- 部分接受会保留未选择的原始段落。
- 保存提示词写入独立个人提示词库，不写入模板库。
- Markdown、HTML、JSON、TXT 导出成功；PDF 路径打开可打印 HTML 页面并保留代码块样式。
- 本地历史开关和敏感信息拦截路径可用。

对应自动化命令：

```bash
npm run test:e2e
```
