# AI Prompt Enhancer 隐私政策

最后更新：2026-07-14

AI Prompt Enhancer 是一个浏览器端提示词增强扩展。我们将隐私与用户控制作为核心设计原则。

## 处理哪些数据

- 插件只处理用户显式触发“增强提示词”“保存提示词”或“导出对话”的文本。
- 插件不默认读取整个网页，不默认记录所有输入内容，也不会在用户未触发时自动上传输入框内容。
- 绿色小球只用于识别当前可编辑输入框并提供入口；增强动作必须由用户点击或快捷键触发。

## 提示词增强与第三方 AI Provider

- 当用户点击增强时，当前提示词会发送给用户在设置中选择并配置的 AI Provider，例如 OpenAI、Anthropic、Google Gemini、Azure OpenAI、Ollama 或用户自定义兼容端点。
- 发送内容包含用户当前要优化的文本、所选 Skill/优化模式以及必要的模型参数。
- 插件不运营自己的后端服务，不会把提示词发送到开发者服务器。
- 第三方 Provider 对数据的处理由对应服务商的隐私政策和用户自己的账号设置约束。

## API Key 存储

- API Key 默认保存在浏览器本地扩展存储中，并通过本地派生密钥加密保存。
- 插件不会在界面中明文展示已保存的完整 API Key。
- API Key 仅在用户触发对应 Provider 调用时在本地解密并用于请求。

## 历史记录

- 优化历史默认保存在本地，用于用户回溯和对比优化结果。
- 用户可以在设置中关闭历史记录、关闭原始文本保存、清空历史或导出本地数据。
- 当“保存原始文本”关闭时，历史记录不保存原始提示词正文。

## GitHub Skill 导入

- GitHub Skill 导入只读取用户明确指定的仓库、文件或 raw/blob 链接。
- 导入内容按 JSON/YAML Skill 配置解析和校验，不执行远程代码。
- 单文件大小与单次导入数量有限制，以降低误导入和资源滥用风险。

## 对话导出

- 对话导出只在用户点击导出后读取当前页面可见的对话 DOM。
- 用户可以选择 Markdown、HTML、JSON、纯文本或可打印 PDF 页面等格式。
- 导出文件由浏览器本地生成，不会上传到远程服务器。

## 敏感信息保护

- 当敏感信息检测开启时，插件会在本地检查当前输入是否疑似包含密钥、令牌、密码等敏感信息。
- 若检测命中，增强操作会被阻止，避免误发送到第三方 Provider。

## 数据共享与商业用途

- 插件不出售用户数据。
- 插件不使用用户数据进行广告画像。
- 插件不向开发者服务器收集、汇总或训练用户提示词。

## 用户控制

用户可随时在扩展设置中管理 Provider、历史、提示词库、导出数据与界面入口显示方式。
