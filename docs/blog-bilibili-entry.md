# 我做了一个开源的 AI 提示词增强插件：把“不会提问”变成可审阅的工作流

> 适合作为 B 站入站申请的项目介绍文章，也可以作为仓库首页的产品故事。

## 一、我为什么做这个项目

使用 ChatGPT、Claude、Gemini 或 DeepSeek 时，我经常遇到一个问题：我知道自己想解决什么，但写出来的提示词只有一句话。模型当然可以回答，但结果的质量会受到上下文、约束、输出格式和验收标准影响。

于是我做了 **AI Prompt Enhancer**：一个开源的 Chrome 浏览器扩展。它不要求用户离开当前网页，也不直接改掉原始输入，而是在输入框旁边提供一个轻量入口，帮助用户把自然语言想法整理成更完整、更容易执行的 Prompt。

这个项目也是我对“AI 工具应该如何尊重用户控制权”的一次实践：先生成、再对比、最后由用户决定是否采用。

## 二、它能做什么

### 1. 在常用 AI 网页旁边工作

扩展支持 ChatGPT、Claude、Gemini、DeepSeek、Copilot 等站点。输入框旁边会出现一个绿色小球，点击后可以进入增强、保存提示词、打开提示词库和配置 API 等操作。小球会根据输入框位置自动避让，尽量不遮挡用户正在输入的内容。

### 2. 用 Skill 把优化方法变成可复用资产

扩展内置了通用增强、代码生成、代码审查、文档撰写、调试辅助和算法解释等 Prompt Engineering Skill。用户可以选择默认 Skill，也可以从 GitHub 仓库、JSON/YAML 文件或粘贴配置导入自己的 Skill。

每个 Skill 都包含名称、描述、适用标签、作者、来源、版本、提示词模板、变量和触发条件。导入内容只会被解析为配置，不会执行远程脚本。

### 3. 不足信息时先追问

有些请求不是直接改写就能解决的。例如“帮我写一个系统”可能缺少用户角色、技术栈、接口约束和验收方式。扩展提供“追问模式”，会先提出需要补充的问题，用户回答后再生成最终 Prompt，让优化过程更接近真实需求分析。

### 4. 支持多个 AI Provider

项目没有自建后端，用户可以配置自己的 OpenAI、Anthropic、Gemini、Azure OpenAI、Ollama、DeepSeek、Kimi、通义千问、智谱、硅基流动、火山方舟或自定义兼容接口。

我特别关注了第三方 API 的差异：不同 Provider 的认证头、请求格式、响应结构和流式协议并不完全相同。因此项目把 Provider 适配层独立出来，并对 OpenAI-compatible、Anthropic、Gemini 和 Ollama 做了兼容处理。生成时支持流式片段和进度提示，长输出还会标记截断状态，避免用户误以为结果已经完整返回。

### 5. 给用户最后的确认权

增强结果会展示为“原始提示词 / 优化后提示词”对比视图，并提供：

- 接受优化：替换输入框内容；
- 拒绝：关闭对比，保留原文；
- 继续优化：补充“再简洁一些”“增加错误处理示例”等要求；
- 部分接受：只采纳满意的段落，其余保留原始内容。

在用户点击“接受优化”以前，原输入框不会被直接替换。这是项目最重要的交互约束之一。

### 6. 个人提示词库和对话导出

用户可以把输入框内容、选中文本、对话消息或增强结果保存到独立的个人提示词库，并添加标签、分类和备注。当前页面可见对话还可以导出为 Markdown、HTML、JSON、TXT，PDF 则通过可打印 HTML 页面生成，并保留代码块样式和对话层级。

## 三、技术实现

项目采用 Chrome Manifest V3 + React + TypeScript + Vite，主要分为四层：

```text
网页输入框
    ↓
Content Script：定位输入框、浮球、对比视图、导出
    ↓ chrome.runtime 消息
Service Worker：Provider 调用、权限检查、流式片段转发、历史保存
    ↓
Local Storage：设置、Skill、个人提示词库、历史、加密后的 API Key
```

几个实现原则：

1. **显式触发**：默认不读取整个网页，也不自动上传所有输入；增强、保存和导出都由用户主动触发。
2. **权限最小化**：默认只覆盖明确支持的站点，不使用 `<all_urls>`，不请求 `tabs`；其他网站通过可选权限按需启用。
3. **本地优先**：没有开发者后端，历史和个人提示词库默认保存在本地，API Key 使用 Web Crypto 加密保存。
4. **可回归验证**：项目包含单元测试和 Playwright E2E。E2E 使用本地 fixture 与 mock Provider，不调用真实第三方 AI API。

## 四、我如何验证它不是“只能 build 的 Demo”

当前版本已经验证：

- Popup、Options、Side Panel 可以从真实构建产物加载；
- 本地 fixture 输入框能稳定出现绿色入口，并且不会覆盖输入内容；
- 增强进度、对比、拒绝、接受、继续优化和部分接受流程可运行；
- 新输入内容不会误用上一轮缓存；
- 保存提示词确实写入独立个人提示词库；
- Markdown、HTML、JSON、TXT 导出可生成文件；
- PDF 路径可以打开可打印页面并保留代码块样式；
- 32 个测试和完整 typecheck、lint、build、E2E 命令通过。

## 五、隐私和安全

提示词只会发送到用户自己选择并配置的 Provider。项目不接收 API Key，不出售用户数据，也不用于广告画像。用户可以关闭历史、关闭原始文本保存、清空或导出本地数据。启用敏感信息检测后，疑似密钥、令牌和密码会在本地被阻止发送。

这并不意味着第三方 Provider 不会处理请求内容：用户仍应阅读所选服务商的隐私政策，并避免把不应发送的数据放入提示词中。
